# Phase 1 - Foundation — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Upstream decisions from 01-CONTEXT.md (D-01 through D-15) are baked into this contract. Discretionary decisions (spacing, typography, color, copy, registry) are declared below.

---

## Design System

| Property          | Value                               |
| ----------------- | ----------------------------------- |
| Tool              | shadcn/ui                           |
| Preset            | slate                               |
| Component library | Radix UI (via shadcn)               |
| Icon library      | lucide-react                        |
| Font              | Geist Sans (Next.js 16 default)     |

### shadcn Initialization Notes

- shadcn/ui will be initialized during execution with `npx shadcn@latest init --yes --template next --base-color slate`
- Post-init, the following components MUST be installed:
  - `button`, `card`, `dialog`, `table`, `input`, `label`, `form`, `badge`, `skeleton`, `tabs`, `textarea`, `dropdown-menu`, `sidebar`, `sheet`, `command`, `pagination`, `select`, `separator`, `avatar`, `tooltip`, `scroll-area`
- Toast notifications use **Sonner** (`npx shadcn add sonner`) — replaces legacy `toast` component

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage                                |
| ----- | ----- | ------------------------------------ |
| xs    | 4px   | Icon gaps, inline padding            |
| sm    | 8px   | Compact element spacing              |
| md    | 16px  | Default element spacing              |
| lg    | 24px  | Section padding                      |
| xl    | 32px  | Layout gaps                          |
| 2xl   | 48px  | Major section breaks                 |
| 3xl   | 64px  | Page-level spacing                   |

**Exceptions:**
- Icon-only touch targets (mobile nav buttons, close buttons): **44px minimum** to meet WCAG 2.5.5
- Admin sidebar item height: **40px** for comfortable click targets
- Upload dropzone min-height: **200px** (visual prominence)

## Typography

| Role    | Size | Weight | Line Height | Usage                              |
| ------- | ---- | ------ | ----------- | ---------------------------------- |
| Body    | 16px | 400    | 1.5         | Paragraphs, descriptions, labels   |
| Label   | 14px | 400    | 1.4         | Metadata, captions, table cells    |
| Heading | 20px | 600    | 1.2         | Page titles, section headers       |
| Display | 28px | 600    | 1.1         | Empty state heading, hero text     |

**Rules:**
- Only 2 font weights used: **400 (normal)** and **600 (semibold)**
- No text smaller than 14px in the UI
- Book titles on cards: Heading size (20px), semibold, clamped to 2 lines with `line-clamp-2`
- Author names on cards: Label size (14px), normal weight, `text-muted-foreground`

## Color

| Role            | Value     | Usage                                          |
| --------------- | --------- | ---------------------------------------------- |
| Dominant (60%)  | `#f8fafc` | Page background, body surface (slate-50)       |
| Secondary (30%) | `#ffffff` | Cards, modals, elevated surfaces               |
| Secondary-alt   | `#f1f5f9` | Sidebar background, nav rail (slate-100)       |
| Accent (10%)    | `#0f172a` | Primary actions, active states, emphasis       |
| Destructive     | `#dc2626` | Destructive actions only (red-600)             |

### Accent Reserved For (Explicit List)

The accent color (`slate-900`) is reserved for:
1. **Primary CTA buttons** — "Upload Book", "Sign In", "Save Changes"
2. **Active sidebar nav item** — background `slate-900`, text white, with left 3px border indicator
3. **Upload dropzone active state** — border turns solid `slate-900`, background `slate-50`
4. **Current step in processing indicator** — filled circle + bold label
5. **Book title text on cards** — `slate-900` (near-black for maximum readability)
6. **Selected tab underline** — 2px `slate-900` bar
7. **Page heading text** — `slate-900`

### Semantic Colors

| Token              | Value     | Usage                          |
| ------------------ | --------- | ------------------------------ |
| Muted foreground   | `#64748b` | Secondary text, placeholders   |
| Border             | `#e2e8f0` | Dividers, card borders         |
| Success            | `#16a34a` | Toast success, completed step  |
| Warning            | `#ca8a04` | Cautionary states              |

### Cover Placeholder Colors

When no EPUB cover is available, generate a solid background from the book title hash mapped to the following slate/blue palette:
- `#cbd5e1` (slate-300), `#94a3b8` (slate-400), `#64748b` (slate-500), `#475569` (slate-600), `#334155` (slate-700), `#1e293b` (slate-800)
- Overlay a `BookOpen` lucide icon at 40% white opacity, centered.

## Component Patterns

### Library Grid Card

- Width: responsive — `minmax(160px, 1fr)` on mobile, `minmax(200px, 1fr)` on tablet+
- Gap: `lg` (24px)
- Card structure:
  - Cover area: 3:4 aspect ratio, rounded `md` (8px), overflow hidden
  - Title: Heading size, semibold, `line-clamp-2`, top margin `sm` (8px)
  - Author: Label size, muted foreground
  - Language badge (if enabled): absolute top-right of cover, `xs` padding, `sm` text, white bg with shadow

### Empty State

- Centered vertically and horizontally in the content area
- Icon: `BookOpen` at 48px, muted foreground color
- Heading: Display size (28px), semibold
- Body: Body size (16px), muted foreground, max-width 400px, centered
- CTA: Primary button "Upload Book", margin-top `lg` (24px)

### Upload Dropzone

- Border: 2px dashed `border` color, rounded `lg` (8px)
- Min-height: 200px
- Centered content: `Upload` icon (32px), dropzone label (Heading size), sublabel (Label size, muted)
- Active state (file dragged over): solid border accent color, background `slate-50`
- Error state (wrong file type): border `destructive`, text `destructive`
- Processing overlay: semi-transparent white backdrop + vertical step indicator

### Processing Step Indicator

- Vertical list of 4 steps
- Each step: circle (20px) + label (Label size)
- Completed step: green checkmark inside circle, label muted
- Current step: filled accent circle, label accent + semibold
- Pending step: empty circle with border, label muted

### Loading Skeletons

- **Library grid**: 8 card skeletons matching grid layout
  - Each skeleton: rounded rectangle (3:4 aspect) + 2 text lines (60% and 40% width)
- **Book detail page**: 
  - Cover skeleton: 3:4 aspect, rounded `md`
  - Title skeleton: 70% width, Heading height
  - Author skeleton: 40% width, Label height
  - Content area skeleton: full width, 200px height
- **Admin tables**: 5 row skeletons, each matching table column widths
- Skeleton color: `bg-muted` (slate-100), pulse animation

### Toast / Sonner

- Position: bottom-right, offset 24px from edges
- Success: green left border, checkmark icon, 4000ms duration
- Error: red left border, alert-circle icon, 6000ms duration
- Info: accent left border, info icon, 4000ms duration
- Dismissible via click or auto-dismiss

### Admin Sidebar

- Desktop: fixed 260px width, `Secondary-alt` background, collapsible
- Mobile: Sheet overlay, triggered by hamburger in admin header
- Nav items (top to bottom):
  1. **Users** — `Users` icon — `/admin/users`
  2. **Universal Library** — `Library` icon — `/admin/books`
  3. **Prompt Templates** — `FileText` icon — `/admin/prompts`
  4. **Audit Log** — `ScrollText` icon — `/admin/audit`
- Active item: accent background, white text, 3px left border accent
- Inactive item: transparent bg, muted foreground text
- Item height: 40px, padding `md` horizontal, `sm` vertical
- Icon size: 18px, gap `sm` (8px) to label

### Admin Tables

- Default sort: `createdAt` descending
- Default page size: 20 rows
- Pagination: shadcn Pagination component at table bottom
- Search: Input with `Search` icon, debounced 300ms
- Column headers: Label size, semibold, clickable for sort (where applicable)
- Row hover: `slate-50` background

### Prompt Template Editor

- Two tabs: "Book-Level" and "Section-Level"
- Each tab: `textarea` with `lg` (24px) padding, monospace font for prompt content
- Word count display below textarea (Label size, muted)
- Save Template button: primary, disabled when no changes
- Discard Changes button: secondary, reverts to last saved

### Profile Page

- Avatar (from Google OAuth `image`) + fallback initials
- Display name and email
- Role badge: pill shape —
  - Regular: `secondary` variant
  - Pro: `default` variant (accent bg)
  - Admin: `outline` variant with `Shield` icon prefix
- "Upgrade to Pro" text for Regular users (disabled/grayed, no click action)

## Accessibility Baseline

- All interactive elements meet **44px minimum touch target**
- Color contrast ratios: accent on white = 15.8:1 (AAA), body on dominant = 12.1:1 (AAA)
- Focus rings: 2px `slate-900` outline with 2px offset on all focusable elements
- Loading states: skeletons have `aria-busy="true"`, reduced-motion respect via `prefers-reduced-motion`
- Empty states: descriptive text + clear CTA, no dead ends
- Upload zone: keyboard-accessible via hidden file input + button trigger
- Icon-only controls (mobile nav hamburger, Sheet/Dialog close buttons, avatar dropdown trigger): must include descriptive `aria-label` and visible `Tooltip` on hover/focus
