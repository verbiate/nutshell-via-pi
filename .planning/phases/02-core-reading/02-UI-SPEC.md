## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                 |
| ----- | ----- | ------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding             |
| sm    | 8px   | Toolbar button gaps, ToC row padding  |
| md    | 16px  | ToC panel header padding, card gaps   |
| lg    | 24px  | Section breaks in ToC list            |
| xl    | 32px  | Reader chrome horizontal padding      |
| 2xl   | 48px  | Error/loading state vertical spacing  |
| 3xl   | 64px  | Page-level spacing                    |

**Reader-specific exceptions:**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| Reader chrome height | `h-12` (48px) | Minimal, non-distracting top bar |
| ToC panel width (mobile) | `w-[320px]` | Fits iPhone 14 Pro Max with safe area |
| ToC panel width (desktop) | `sm:w-[360px]` | Comfortable 40-character line for chapter titles |
| ToC indent step | `16px` per level | Visual hierarchy without excessive nesting width |
| Bottom progress bar | `h-1` (4px) | Subtle, non-distracting position indicator |
| EPUB iframe container | `inset-0` (no padding) | Content fills entire viewport below chrome |

## Typography

### Reader Chrome (design system controlled)

| Role    | Size   | Weight   | Line Height | Usage                          |
| ------- | ------ | -------- | ----------- | ------------------------------ |
| Body    | 14px   | 400      | 1.5         | ToC entries, tooltips          |
| Label   | 12px   | 500      | 1.4         | Progress percentage, badges    |
| Heading | 16px   | 500      | 1.3         | ToC panel title (SheetTitle)   |
| Display | 14px   | 500      | 1.4         | Book title in chrome, truncated|

### EPUB Content (iframe controlled, NOT design system)

The EPUB renders inside an iframe managed by `@likecoin/epub-ts`. Typography for book content is injected via `rendition.themes.register()` and is intentionally separate from the app design system. The following values are the **injected CSS** for each theme:

| Theme  | Body Font Size | Line Height | Font Family              |
|--------|----------------|-------------|--------------------------|
| Light  | 18px           | 1.6         | system-ui, Georgia, serif|
| Dark   | 18px           | 1.6         | system-ui, Georgia, serif|
| Sepia  | 18px           | 1.6         | system-ui, Georgia, serif|

**Rationale:** 18px/1.6 is the industry standard for ebook readability (matches Kindle, Apple Books). Georgia provides excellent screen readability for long-form text. The app chrome uses Geist (sans-serif) for UI distinction from content.

## Layout Contract

### Focal Point
**Primary focal point:** Book content (EPUB iframe) fills the entire viewport (`inset-0`). Reader chrome is intentionally subordinate — a translucent `h-12` toolbar at top and `h-1` progress bar at bottom. The eye is drawn to the reading material, not the controls.

### Route Structure

```
src/app/
  (reader)/
    layout.tsx          # Full-screen, no header, no footer
    book/[id]/
      reader/
        page.tsx        # Reader page component
```

### Reader Layout Spec

| Element | Position | Dimensions | Z-Index | Styling |
|---------|----------|------------|---------|---------|
| EPUB container | Absolute, inset-0 | 100vw x 100vh | 0 | `bg-background` |
| Chrome toolbar | Absolute, top-0 | Full width x h-12 | 50 | `bg-background/80 backdrop-blur-sm border-b border-border/50` |
| Bottom progress | Absolute, bottom-0 | Full width x h-1 | 50 | `bg-muted` track, `bg-primary` fill |
| ToC Sheet | Fixed, left side | w-[320px] sm:w-[360px] | 100 | shadcn Sheet with `side="left"` |
| Loading overlay | Absolute, inset-0 | Full viewport | 60 | `bg-background` with centered skeleton |
| Error overlay | Absolute, inset-0 | Full viewport | 60 | Centered card with actions |

### Toolbar Layout (left to right)

```
[Back btn] [ToC btn] ---- [Book title, truncated, centered] ---- [Theme btn]
```

- Back button: `variant="ghost"`, `size="icon-sm"`, `ChevronLeft` icon
- ToC button: `variant="ghost"`, `size="icon-sm"`, `Menu` icon
- Book title: `text-sm font-medium truncate max-w-[200px] sm:max-w-[400px] text-foreground`
- Theme toggle: `variant="ghost"`, `size="icon-sm"`, cycles Sun → BookOpen → Moon

```tsx
<button
  className={cn(
    "w-full text-left py-2 pr-4 text-sm transition-colors",
```

### 3. ThemeToggle (`src/components/reader/theme-toggle.tsx`)

```tsx
// Spec (pseudocode)
const cycleTheme = () => {
  if (resolvedTheme === 'light') setTheme('sepia');
  else if (resolvedTheme === 'sepia') setTheme('dark');
  else setTheme('light');
};

<Button variant="ghost" size="icon-sm" onClick={cycleTheme} aria-label="Cycle theme">
  {resolvedTheme === 'light' && <Sun className="h-4 w-4" />}
  {resolvedTheme === 'sepia' && <BookOpen className="h-4 w-4" />}
  {resolvedTheme === 'dark' && <Moon className="h-4 w-4" />}
</Button>
```

**Behavior:**
- Cycles light → sepia → dark → light
- Must be mount-gated: `const [mounted, setMounted] = useState(false); useEffect(() => setMounted(true), []);`
- Before mount, render `<div className="h-7 w-7" />` placeholder (same dimensions as icon-sm button)
- Calls `rendition.themes.select(theme)` via prop callback when theme changes

## Theme Contract

### next-themes Configuration

```tsx
// In src/components/providers.tsx
import { ThemeProvider } from 'next-themes';

<ThemeProvider
  attribute="class"
  defaultTheme="light"
  themes={['light', 'dark', 'sepia']}
  enableSystem={false}
  disableTransitionOnChange={false}
>
  {children}
</ThemeProvider>
```

**Rules:**
- `enableSystem: false` — user must explicitly choose; no system preference override
- `disableTransitionOnChange: false` — allow CSS transitions for smooth theme switches
- `attribute="class"` — adds `class="sepia"` to `<html>` when sepia is active

### Dual Theme Sync (App + Iframe)

| Event | App Action | Iframe Action |
|-------|-----------|---------------|
| Reader mounts | — | Register light/dark/sepia themes on rendition |
| User clicks theme toggle | `setTheme(next)` | `rendition.themes.select(next)` |
| `resolvedTheme` changes | HTML class updates | CSS injected into iframe body |

**Critical:** Both systems must update simultaneously. Never update only the app shell or only the iframe.

## Interaction Patterns

### Keyboard

| Key | Action |
|-----|--------|
| `Escape` | Close ToC panel if open; otherwise navigate back to library |
| `ArrowLeft` | Previous page (paginated mode) |
| `ArrowRight` | Next page (paginated mode) |
| `t` | Cycle theme |

### Mouse / Touch

| Action | Result |
|--------|--------|
| Click ToC trigger | Open ToC Sheet |
| Click ToC entry | Navigate to section, close Sheet |
| Click theme button | Cycle light → sepia → dark |
| Click back button | Navigate to `/my-library` |
| Swipe left / right | **Deferred to v2** (no swipe gestures in v1) |

### Accessibility

- All toolbar buttons have explicit `aria-label`
- ToC entries are `<button>` elements (not divs)
- Sheet has `aria-labelledby` pointing to SheetTitle
- EPUB iframe has `aria-label="Book content"`
- Focus trap handled by Radix Dialog (Sheet primitive)
