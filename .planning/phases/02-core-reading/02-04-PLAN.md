  <name>Task 1: Build position tracking library (CFI ↔ paragraph mapping)</name>
  <files>src/lib/reader/position-tracking.ts</files>
  <action>
    Create src/lib/reader/position-tracking.ts:

    Types:
    - ReadingPosition = { paragraphIndex: number; charOffset: number; cfi?: string }
    - ParagraphMap = Array&lt;{ spineIndex: number; sectionHref: string; paragraphCount: number; paragraphCfis: string[] }&gt;

    Functions:

    1. buildParagraphMap(book: Book): Promise&lt;ParagraphMap&gt;
       - Iterate book.spine.spineItems
       - For each section, load section.render() to get HTML document
       - Count all &lt;p&gt; elements: doc.querySelectorAll('p')
       - For each paragraph, generate its CFI using EpubCFI.generateChapterComponent()
       - Store: { spineIndex, sectionHref, paragraphCount, paragraphCfis: string[] }
       - Return array

    2. cfiToParagraphOffset(book: Book, cfi: string, paragraphMap: ParagraphMap): ReadingPosition
       - Use book.spine.get(cfi) to find current Section
       - Use EpubCFI.parse(cfi) + EpubCFI.findNode() to locate DOM element and text offset inside iframe
       - Find section in paragraphMap by spineIndex
       - Count &lt;p&gt; elements in preceding spine items (sum paragraphMap[i].paragraphCount for i &lt; currentSpineIndex)
       - Count &lt;p&gt; elements within current section up to target element
       - Return { paragraphIndex: precedingCount + currentCount, charOffset: textOffsetWithinParagraph, cfi }

    3. paragraphOffsetToCfi(book: Book, position: ReadingPosition, paragraphMap: ParagraphMap): string
       - Find spine item containing paragraphIndex by accumulating paragraphMap[].paragraphCount
       - Load target section, find Nth &lt;p&gt; element
       - Construct CFI for that paragraph at charOffset using EpubCFI.generateChapterComponent() + path to paragraph
       - Return CFI string

    4. getSectionForParagraph(paragraphMap: ParagraphMap, paragraphIndex: number): { spineIndex: number; sectionHref: string }
       - Accumulate counts to find which spine item contains the paragraph

    Export all functions. Use @likecoin/epub-ts imports (EpubCFI, Book).
  </action>

  </verify>

  <name>Task 2: Build server service and API route for position CRUD</name>
  <files>src/server/services/reader.ts, src/app/api/reader/position/route.ts</files>
  <action>
    Create src/server/services/reader.ts:

    Functions:
    1. getPosition(userId: string, bookId: string): Promise&lt;UserBookPosition | null&gt;
       - prisma.userBookPosition.findUnique({ where: { userId_bookId: { userId, bookId } } })

    2. savePosition(userId: string, bookId: string, data: { paragraphIndex: number; charOffset: number; cfi?: string; tocSectionId?: string }): Promise&lt;UserBookPosition&gt;
       - prisma.userBookPosition.upsert({
           where: { userId_bookId: { userId, bookId } },
           create: { userId, bookId, ...data },
           update: { ...data }
         })

    3. verifyBookAccess(userId: string, bookId: string): Promise&lt;boolean&gt;
       - Check if user has access via library service or direct prisma query on UserBookAccess
       - Return true if access exists

    Create src/app/api/reader/position/route.ts:

    GET handler:
    - Authenticate via requireAuth()
    - Parse searchParams: bookId
    - Call verifyBookAccess(session.user.id, bookId) — return 403 if denied
    - Call getPosition(session.user.id, bookId)
    - Return JSON: { position: { paragraphIndex, charOffset, cfi, tocSectionId } | null }

    POST handler:
    - Authenticate via requireAuth()
    - Parse JSON body: bookId, paragraphIndex, charOffset, cfi?, tocSectionId?
    - Validate: paragraphIndex >= 0, charOffset >= 0, bookId is non-empty string
    - Call verifyBookAccess(session.user.id, bookId) — return 403 if denied
    - Call savePosition(session.user.id, bookId, { paragraphIndex, charOffset, cfi, tocSectionId })
    - Return JSON: { success: true }
  </action>
