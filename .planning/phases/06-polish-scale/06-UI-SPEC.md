## Layout Contract

### Focal Point
**Primary focal point:** The bookshelf grid of book covers. The progress bar is intentionally subordinate — a thin line at the bottom edge of each cover, visible at a glance but never competing with the cover art. Pro badges are status indicators in the header, not focal elements.

```
src/app/
  (library)/
    layout.tsx              # Library header with RoleBadge + UserNav
    my-library/
      page.tsx              # Server page: fetches library + progress, renders Bookshelf

```

## Component Specifications

### 1. BookCard (`src/components/library/book-card.tsx`)

**Props interface:**
```tsx
interface BookCardProps {
  id: string;
  title: string;
  author: string | null;
  language: string;
  coverPath: string | null;
  progress?: number | null; // NEW: 0-100, null/undefined = never opened
}
```

**Spec (pseudocode):**
```tsx
<Link href={`/book/${id}`} className="group block">
  <div className="overflow-hidden rounded-md transition-shadow duration-200 hover:shadow-md">
    <div className="relative aspect-[3/4] w-full bg-slate-100">
      {coverPath ? (
        <img
          src={`/api/files/covers/${id}.jpg`}
          alt={title}
          className="h-full w-full object-cover transition-transform duration-300 group-hover:scale-105"
        />
      ) : (
        <div
          className="flex h-full w-full items-center justify-center bg-gradient-to-br from-white/10 to-black/10"
          style={{ backgroundColor: bgColor }}
        >
          <BookOpen className="h-12 w-12 text-white/40" />
        </div>
      )}
      {language && language !== "und" && (
        <span className="absolute right-1.5 top-1.5 rounded bg-white/90 px-1.5 py-0.5 text-xs font-medium text-slate-700 shadow-sm">
          {language.toUpperCase()}
        </span>
      )}
      {progress !== undefined && progress > 0 && (
        <>
          {/* Progress bar track */}
          <div
            className="absolute bottom-0 left-0 right-0 h-1 bg-black/10"
            role="progressbar"
            aria-valuenow={Math.round(progress)}
            aria-valuemin={0}
            aria-valuemax={100}
            aria-label={`Reading progress: ${Math.round(progress)}%`}
          >
            {/* Progress bar fill */}
            <div
              className="h-full bg-primary transition-all duration-300"
              style={{ width: `${progress}%` }}
            />
          </div>
          {/* Percentage label */}
          <span className="absolute bottom-1.5 right-1.5 text-[10px] font-medium text-white drop-shadow-sm">
            {Math.round(progress)}%
          </span>
        </>
      )}
    </div>
    <div className="px-1 pt-2">
      <h3 className="line-clamp-2 text-base font-semibold leading-tight text-slate-900">
        {title}
      </h3>
      {author && (
        <p className="mt-1 truncate text-sm text-muted-foreground">
          {author}
        </p>
      )}
    </div>
  </div>
</Link>
```

**Behavior:**
- `group` class on `Link` enables `group-hover` on child elements
- Cover image scales to `105%` on hover with `duration-300` ease
- Card container gains `shadow-md` on hover with `duration-200` ease
- Progress bar only renders when `progress > 0` (D-02: books never opened show no progress bar)
- Progress bar uses the same `h-1 bg-primary transition-all duration-300` pattern as the reader's `ReadingProgress` component
- Percentage label overlays the cover at bottom-right with white text + drop-shadow for legibility on any cover
- Title reduced to `text-base` (16px) for better proportions at 200px card width
- Placeholder covers gain a subtle `bg-gradient-to-br from-white/10 to-black/10` overlay for depth

## Interaction Patterns

### Hover States

| Element | Trigger | Effect | Duration | Easing |
|---------|---------|--------|----------|--------|
| BookCard container | `group-hover` | `shadow-md` | 200ms | default (ease) |
| BookCard cover image | `group-hover` | `scale-105` | 300ms | default (ease) |
| Library nav links | hover | `text-slate-900` | 150ms | default |
| RoleBadge (Admin/Pro) | hover | No specific hover effect required | — | — |

### Progress Bar Animation
- Fill width animates with `transition-all duration-300` when progress value changes
- On initial render, bar appears at final width (no entrance animation required)
- Track is always present when progress is shown; only fill width changes

### Accessibility

- Progress bar has `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label` (mirrors `ReadingProgress` pattern)
- Percentage label is a visual enhancement; the aria-label on the progressbar provides the accessible value
- BookCard `Link` is a semantic anchor element with href (inherited)
- RoleBadge uses semantic `span` via shadcn Badge (inherited)
- All hover effects are visual-only and do not affect keyboard navigation

## Appendix: Data Flow for Progress Indicators

This section is informational for implementers and does not affect the UI contract.

1. `EpubFile` model gains `totalParagraphs Int?` (computed at upload time from TXT split)
2. `getPersonalLibrary(userId)` fetches `UserBookAccess` + `UserBookPosition` in parallel
3. Progress computed server-side: `% = (paragraphIndex / totalParagraphs) * 100`
4. `progress: number | null` added to library book type (null = never opened or missing totalParagraphs)
5. `my-library/page.tsx` passes `progress` to `Bookshelf` → `BookCard`
6. `BookCard` renders progress bar when `progress > 0`

For existing books without `totalParagraphs`, progress bar does not render (graceful degradation).
