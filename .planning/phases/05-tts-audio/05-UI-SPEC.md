## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                 |
| ----- | ----- | ------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding             |
| sm    | 8px   | Toolbar button gaps, player controls  |
| md    | 16px  | Player horizontal padding, card gaps  |
| lg    | 24px  | Section breaks, admin form spacing    |
| xl    | 32px  | Layout gaps, admin page header        |
| 2xl   | 48px  | Major section breaks                  |
| 3xl   | 64px  | Page-level spacing                    |

**Phase 5-specific exceptions:**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| TTS bottom bar height | `h-16` (64px) | Compact but touch-friendly player chrome |
| TTS bottom bar expanded padding-bottom | `pb-16` | EPUB viewer shrinks so bar never overlaps text |
| Progress scrubber track height | `h-1` (4px) | Matches reading progress bar aesthetic |
| Admin config row card | `p-4` (16px) | Consistent with admin table density |

## Layout Contract

### Focal Point
**Primary focal point:** Book content (EPUB iframe) remains the focal point. The TTS bottom bar is subordinate — it slides up only during active playback and collapses away when idle. The trigger button in the chrome toolbar is a single unobtrusive icon.

### Route Structure

```
src/app/
  (reader)/
    book/[id]/
      reader/
        page.tsx        # Reader page — TtsPlayer injected here
  admin/
    config/
      page.tsx          # NEW: Admin API Keys & Models config
```

### Reader Layout Spec (Updated for TTS)

| Element | Position | Dimensions | Z-Index | Styling |
|---------|----------|------------|---------|---------|
| EPUB container | Absolute, inset-0 | 100vw x 100vh | 0 | `bg-background`, `pb-16` when player active |
| Chrome toolbar | Absolute, top-0 | Full width x h-12 | 50 | `bg-background/80 backdrop-blur-sm` |
| TTS bottom bar | Fixed, bottom-0 | Full width x h-16 | 60 | `bg-background/95 backdrop-blur-sm border-t border-border` |
| Bottom progress | Absolute, bottom-0 | Full width x h-1 | 50 | `bg-muted` track, `bg-primary` fill |

**Critical:** When TTS player is active, EPUB container must gain `pb-16` so the bottom bar never obscures text. When player is idle/collapsed, `pb-0`.

### TTS Bottom Bar Layout (left to right)

```
[Play/Pause/Spinner] [Section label, truncated] [Progress scrubber] [Duration] [Close]
```

- Play/Pause button: `variant="ghost"`, `size="icon-sm"`, `Play` / `Pause` icon
- Spinner (generating): `Loader2` with `animate-spin`, same size as icon-sm
- Section label: `text-sm font-medium truncate max-w-[200px] sm:max-w-[300px] text-foreground`
- Progress scrubber: shadcn Slider, `h-1` track, full remaining width
- Duration: `text-xs text-muted-foreground`, format `MM:SS / MM:SS`
- Close button: `variant="ghost"`, `size="icon-sm"`, `X` icon

## Component Specifications

### 1. TtsTrigger (`src/components/reader/tts-trigger.tsx`)

```tsx
// Spec (pseudocode)
<Button
  variant="ghost"
  size="icon-sm"
  onClick={onClick}
  disabled={disabled}
  aria-label={isGenerating ? "Cancel audio generation" : "Read aloud"}
>
  {isGenerating ? <Loader2 className="h-4 w-4 animate-spin" /> : <Volume2 className="h-4 w-4" />}
</Button>
```

**States:**
- **Idle:** `Volume2` icon, enabled, tooltip "Start reading aloud"
- **Generating:** `Loader2` spinning icon, enabled (not disabled — re-click cancels), tooltip "Cancel audio generation"
- **Disabled:** `Volume2` icon, `disabled={true}`, tooltip "Ask your admin to configure TTS" (when TTS not configured for user's tier)

**Integration:** Add `ttsTrigger?: ReactNode` to `ReaderChromeProps` and place it in the right group, between `searchTrigger` and `themeToggle`:

```tsx
// Updated ReaderChrome right group
<div className="flex items-center gap-1 shrink-0">
  {searchTrigger}
  {ttsTrigger}
  {themeToggle}
</div>
```

### 2. TtsPlayer (`src/components/reader/tts-player.tsx`)

```tsx
// Spec (pseudocode)
<div
  className={cn(
    "fixed bottom-0 left-0 right-0 z-60 h-16 flex items-center gap-3 px-4 border-t border-border bg-background/95 backdrop-blur-sm transition-transform duration-300",
    visible ? "translate-y-0" : "translate-y-full"
  )}
>
  {/* Play / Pause / Spinner */}
  <Button variant="ghost" size="icon-sm" onClick={handlePlayPause}>
    {state === "GENERATING" && <Loader2 className="h-4 w-4 animate-spin" />}
    {state === "PLAYING" && <Pause className="h-4 w-4" />}
    {(state === "IDLE" || state === "READY" || state === "ENDED") && <Play className="h-4 w-4" />}
  </Button>

  {/* Section label */}
  <span className="text-sm font-medium truncate max-w-[200px] sm:max-w-[300px] text-foreground shrink-0">
    {sectionTitle}
  </span>

  {/* Progress scrubber */}
  <div className="flex-1 min-w-0">
    <Slider
      value={[progress]}
      max={duration || 100}
      step={1}
      onValueChange={handleScrub}
      className="w-full"
    />
  </div>

  {/* Duration */}
  <span className="text-xs text-muted-foreground tabular-nums shrink-0">
    {formatTime(currentTime)} / {formatTime(duration)}
  </span>

  {/* Close */}
  <Button variant="ghost" size="icon-sm" onClick={handleClose} aria-label="Close audio player">
    <X className="h-4 w-4" />
  </Button>

  {/* Hidden audio element */}
  <audio ref={audioRef} onEnded={handleEnded} onTimeUpdate={handleTimeUpdate} />
</div>
```

**State machine (locked D-03):**

```
IDLE → GENERATING → READY → PLAYING → ENDED → (auto-advance) → GENERATING ...
  ↑                                              ↓
  └──────────────── Close / Cancel ──────────────┘
```

| State | Visual | Behavior |
|-------|--------|----------|
| IDLE | Bar hidden (`translate-y-full`) | No audio activity. Triggered by TtsTrigger click. |
| GENERATING | Bar visible, spinner icon, label shows "Generating audio..." | POST to `/api/tts/generate` in flight. Re-click cancels (AbortController abort). |
| READY | Bar visible, play icon | Audio URL received, about to auto-play. Transitions to PLAYING immediately. |
| PLAYING | Bar visible, pause icon, progress scrubber live | Audio playing. Progress updates via `timeupdate`. Scrubbing seeks. |
| ENDED | Bar visible, play icon, "Loading next section..." brief label | Section finished. Auto-advance to next section (TTS-01). |

**Progress scrubber spec:**
- Track: `h-1 bg-muted`
- Fill: `bg-primary`
- Thumb: `h-3 w-3 bg-primary rounded-full` (shadcn Slider default)
- Disabled during GENERATING state

## Interaction Patterns

### TTS Playback

| Interaction | Result |
|-------------|--------|
| Click TTS trigger (idle) | Start generation for current section, bottom bar slides up, spinner shown |
| Click TTS trigger (generating) | Cancel generation (AbortController), bar returns to IDLE |
| Click Play (ready/ended) | `audio.play()`, transitions to PLAYING |
| Click Pause (playing) | `audio.pause()`, stays in PLAYING state but paused |
| Scrub progress bar | `audio.currentTime = value`, updates UI immediately |
| Section ends | Auto-advance: fetch next section audio, bar shows "Loading next section...", play when ready |
| Click Close | `audio.pause()`, `audio.src = ""`, bar slides down, return to IDLE |

### Keyboard

| Key | Action |
|-----|--------|
| `Space` (when player focused) | Toggle play/pause |
| `Escape` (when player visible) | Close player |

### Accessibility

- TTS trigger has explicit `aria-label`
- Player bar has `role="region"` and `aria-label="Audio player"`
- Progress scrubber uses shadcn Slider (Radix primitive, fully accessible)
- Play/Pause button toggles `aria-pressed`
- Close button has `aria-label="Close audio player"`
- Audio element is hidden (`className="hidden"`) but programmatically controlled
