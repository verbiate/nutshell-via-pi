Append two new models to `src/server/db/schema.prisma` after the `UserBookPosition` model:

```prisma
model Bookmark {
  id             String   @id @default(cuid())
  userId         String
  bookId         String
  cfi            String
  paragraphIndex Int
  charOffset     Int
  selectedText   String?
  note           String?
  createdAt      DateTime @default(now())

  user User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  book EpubFile @relation(fields: [bookId], references: [id], onDelete: Cascade)

  @@unique([userId, bookId, cfi])
  @@index([userId])
  @@index([bookId])
}

model Highlight {
  id             String   @id @default(cuid())
  userId         String
  bookId         String
  cfi            String
  paragraphIndex Int
  charOffsetStart Int
  charOffsetEnd   Int
  selectedText   String
  color          String   @default("#fbbf24")
  note           String?
  createdAt      DateTime @default(now())

  user User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  book EpubFile @relation(fields: [bookId], references: [id], onDelete: Cascade)

  @@unique([userId, bookId, cfi])
  @@index([userId])
  @@index([bookId])
}
```

Also add the `ExplainerRequest` model after the `Explainer` model:

```prisma
model ExplainerRequest {
  id          String   @id @default(cuid())
  userId      String
  bookId      String
  explainerId String
  passageCfi  String?
  passageText String?
  sectionHref String?
  createdAt   DateTime @default(now())

  user      User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  book      EpubFile  @relation(fields: [bookId], references: [id], onDelete: Cascade)
  explainer Explainer @relation(fields: [explainerId], references: [id], onDelete: Cascade)

  @@index([userId, bookId])
  @@index([createdAt])
}
```

Add reverse relations to the `User` model:
- `bookmarks Bookmark[]`
- `highlights Highlight[]`
- `explainerRequests ExplainerRequest[]`

Add reverse relations to the `EpubFile` model:
- `bookmarks Bookmark[]`
- `highlights Highlight[]`
- `explainerRequests ExplainerRequest[]`

Add reverse relation to the `Explainer` model:
- `requests ExplainerRequest[]`
</action>

<read_first>

<action>
In `src/server/services/explainer.ts`, change the `ExplainerLookup` interface:

```ts
export interface ExplainerLookup {
  contentHash: string;
  language: string;
  contentType: "book" | "section" | "passage";
  tier: "regular" | "pro";
}
```

Change the `GenerateExplainerParams` interface:
```ts
export interface GenerateExplainerParams {
  bookId: string;
  type: "book" | "section" | "passage";
  language: string;
  tier: "regular" | "pro";
  sectionHref?: string;
  passageText?: string;
}
```

In the `generateExplainer` function, add a branch for `type === "passage"` after the existing `type === "book"` and `else` (section) branches. For passage type:
1. If `!passageText`, throw `new Error("passageText is required for passage-level explainer")`
2. Call `buildPassagePrompt(bookId, passageText, language)` (function to be created in 04-02)
3. Use `type` as the promptType argument to `computeContentHash`
</action>
