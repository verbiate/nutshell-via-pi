## Spacing Scale

Inherited from Phase 2. All values are multiples of 4.

| Token | Value | Usage                                 |
| ----- | ----- | ------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding             |
| sm    | 8px   | Toolbar button gaps, ToC row padding  |
| md    | 16px  | Panel header padding, card gaps       |
| lg    | 24px  | Section breaks in explainer text      |
| xl    | 32px  | Page-level spacing                    |
| 2xl   | 48px  | Error/loading state vertical spacing  |
| 3xl   | 64px  | Page-level spacing                    |

**Phase 3 additions:**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| Explainer Sheet width (mobile) | `w-[320px]` | Matches ToC panel for visual symmetry |
| Explainer Sheet width (desktop) | `sm:w-[400px]` | Slightly wider than ToC for long-form AI text readability |
| ToC action button size | `h-6 w-6` (24px) | Small inline icon button, does not increase row height |
| Word animation stagger | `0.04s` per word | Readable pace without feeling sluggish |
| Explainer text max-width | `max-w-none` (full panel) | AI text is not book content; use full panel width |

## Typography

Inherited from Phase 2. Declare only Phase 3 additions.

### Explainer Content (AI-generated text)

| Role    | Size   | Weight   | Line Height | Usage                          |
| ------- | ------ | -------- | ----------- | ------------------------------ |
| Body    | 14px   | 400      | 1.6         | Explainer paragraph text       |
| Heading | 16px   | 500      | 1.3         | Explainer panel title          |
| Label   | 12px   | 500      | 1.4         | Cached badge, status labels    |

**Rationale:** 14px/1.6 is slightly more spacious than chrome text (14px/1.5) because explainer text is long-form content meant to be read, not UI chrome. Same size as ToC entries for consistency, but taller line-height for readability.

## Layout Contract

### Explainer Display: Right-Side Sheet

**Decision:** Right-side Sheet (symmetric with left-side ToC Sheet). This keeps the book content visible while reading the explainer, supports the immersive reading pattern, and reuses the established Sheet primitive.

| Element | Position | Dimensions | Z-Index | Styling |
|---------|----------|------------|---------|---------|
| Explainer Sheet | Fixed, right side | w-[320px] sm:w-[400px] | 100 | shadcn Sheet with `side="right"` |
| Sheet header | Sticky top | Full width | — | `px-4 py-3 border-b border-border` |
| Sheet content | Scrollable | Full height minus header | — | `ScrollArea` with `h-[calc(100vh-56px)]` |
| Language override | Inline with header | Auto width | — | shadcn `Select` with compact styling |

### Book Detail Page Trigger Placement

```
[Open Reader] [Explain this to me]
```

- "Open Reader": `variant="default"` (primary)
- "Explain this to me": `variant="outline"`, `Sparkles` icon, positioned immediately to the right of primary CTA
- Gap between buttons: `gap-2` (8px)

### ToC Panel Trigger Placement

Each `TocEntry` gains an inline action button:

```
[Chapter Title ..............................................] [Sparkles]
```

- Explainer button: `variant="ghost"`, `size="icon-xs"` (24px container, 14px icon)
- Visibility: `opacity-0 group-hover:opacity-100` on desktop; always visible on mobile (`md:opacity-0 md:group-hover:opacity-100`)
- Click stops propagation to prevent navigation when clicking explain

### Profile Modal Layout

```
[Avatar + Name + Email]          [Role Badge]
-----------------------------------------------
Preferred language for Explainers
[English ▼]
-----------------------------------------------
[Cancel] [Save Changes]
```

- Dialog width: `max-w-md` (448px)
- Content padding: `p-6`
- Form gap: `gap-4` (16px)

### 4. ExplainerStream (Streaming Text with Animation)

**Location:** `src/components/explainer/explainer-stream.tsx`

```tsx
// Spec (pseudocode)
function ExplainerStream({ text }: { text: string }) {
  const words = text.split(/(\s+)/); // preserve whitespace
  return (
    <div className="text-sm leading-relaxed text-foreground">
      {words.map((word, i) => (
        <span
          key={`${i}-${word}`}
          className="explainer-word inline"
          style={{ '--word-index': i } as React.CSSProperties}
        >
          {word}
        </span>
      ))}
    </div>
  );
}
```

**CSS Animation (add to globals.css):**
```css
@keyframes fadeInWord {
  from {
    opacity: 0;
    transform: translateY(2px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.explainer-word {
  opacity: 0;
  animation: fadeInWord 0.3s ease-out forwards;
  animation-delay: calc(var(--word-index) * 0.04s);
}
```

**Behavior:**
- Text is split on whitespace to preserve word boundaries
- Each word/whitespace chunk gets `--word-index` for stagger
- Animation duration: 0.3s per word
- Stagger delay: 0.04s per word index
- Easing: `ease-out`
- New words appended during streaming animate in automatically
- Maximum animation delay capped at 2s to prevent excessive wait on long text (words beyond index 50 appear with `animation-delay: 2s` or use `opacity: 1` directly)

**Performance note:** For very long explainers (>500 words), switch to batch animation (animate paragraphs as blocks rather than individual words) to prevent DOM thrashing.

## Interaction Patterns

### Explainer Generation Flow

```
User clicks "Explain this to me"
  → Sheet opens on right side
  → Check cache (GET /api/explainers?bookId=X&lang=Y&tier=Z)
    → Cached: display immediately, show "Served from cache" badge
    → Not cached: show loading state, POST to generate
      → SSE stream begins
      → Words animate in one-by-one
      → Stream complete: text persists, badge hidden
```

### Language Override Flow

```
User changes language in explainer Sheet header Select
  → Sheet content resets to loading/empty state
  → New cache check with overridden language
  → Generate if not cached
```

### Keyboard

| Key | Action |
|-----|--------|
| `Escape` | Close explainer Sheet if open |
| `e` | **Deferred to v2** — quick open explainer from reader |

### Mouse / Touch

| Action | Result |
|--------|--------|
| Click book-level "Explain" | Open explainer Sheet, trigger generation |
| Click section-level "Explain" in ToC | Open explainer Sheet with section context |
| Click language Select in Sheet | Dropdown of available languages |
| Click "Try Again" on error | Retry generation with same parameters |
| Click outside Sheet | Close Sheet (standard Sheet behavior) |

### Accessibility

- Explainer Sheet has `aria-labelledby` pointing to SheetTitle
- Streaming text container has `aria-live="polite"` so screen readers announce new content
- Loading state has `role="status"` and `aria-label="Generating explanation"`
- All trigger buttons have explicit `aria-label`
- Language Select has associated `<Label>`
- Focus trap handled by Radix Dialog (Sheet primitive)
