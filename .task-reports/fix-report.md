## Findings addressed

### I1 (Important) — BookCard mirror lifts the whole card → fixed to lift cover only
**Root cause:** `.ds-book-card` lived on the outer `<a>` and the `transform: translateY(...)` was scoped directly to `.ds-book-card:hover`, lifting the entire card (cover + progress slot together). Production `BookCard` only lifts the inner cover wrapper.

**Fix:** Split the hover trigger from the transform target.
- `.ds-book-card` stays on the outer `<a>` (unchanged — still the hover trigger).
- New `.ds-book-lift` class on the inner wrapper `<div>` that wraps the cover.
- CSS: removed `transition`/`transform` from `.ds-book-card` and re-pointed them at `.ds-book-card:hover .ds-book-lift`. The `.ds-book-shadow` rules were left alone (already correctly cascaded via `.ds-book-card:hover .ds-book-shadow`).

Both `demo-1` and `demo-2` tiles updated. Result: hovering anywhere on the `<a>` triggers the lift; only the cover wrapper moves; the progress slot stays anchored — matching `book-card.tsx`.

### M1 (Minor) — Stale `ponytail:` comment → updated
The phrase "Mirrors BookCard's classes verbatim so the shadow reads identically" no longer held (Task 3 replaced `group`/`shadow-book`/`group-hover:shadow-book-lifted`). Rewrote the comment to name the actual mechanism (scoped `.ds-book-card`/`.ds-book-lift`/`.ds-book-shadow` driven by `var(--book-hover-lift)`) while preserving the still-valid explanation of why these are inline mirrors (`BookCard.coverPath` routes through `/api/files/`, which a public asset can't satisfy).

### M2 (Minor) — Hardcoded shadow rgba without `ponytail:` note → documented
Added a one-line `ponytail:` comment immediately above `.ds-book-shadow` naming the duplication (`@utility shadow-book`/`shadow-book-lifted`) and the upgrade path ("re-point at the utilities if group-hover is ever restored"). Did not attempt to reference `@utility` from inside a `<style>` block — not possible; duplication is the correct call given the `group-hover:` removal.

### M3 (Minor) — Empty `className=""` → resolved by I1
Both `<div className="">` wrappers (demo-1 and demo-2) now carry `className="ds-book-lift"`. Verified no other empty `className=""` remains in the two tiles; did not scan elsewhere (out of scope).
